
# HancomEQN 변환

수식이 포함된 이미지를 받아, **텍스트는 그대로 유지**하고 **수식 위치에 HancomEQN으로 변환한 구문을 삽입**한다. 목적은 한컴오피스(HWP)에 그대로 붙여넣어 작업을 쉽게 하는 것이다.

## 출력 형식

- 모든 수식의 시작과 끝에는 '$'를 붙여 구분한다.
  - 예: "{a+b+c+d} over {4} >= root{4} of {abcd}" -> "${a+b+c+d} over {4} >= root{4} of {abcd}$"
- 수식과 텍스트가 **공백 없이 붙어 있으면** 수식 끝에 백틱 문자(')를 붙인다.
  - 예: "b=a이다." → "$b=a`$이다."
- 텍스트(한글 등)는 변환하지 않고 그대로 둔다.
- PDF에 원래 포함되어 있던 내용 외의 설명이나 주석 등은 베제하고 원래 파일의 텍스트와 수식만 포함한다.
  - 예: "이미지에서 수식이 포함된 부분(7. 예시 답안 혹은 정답 섹션)을 변환하겠습니다."와 같은 문장은 작성하지 않는다.

추가로 HancomEQN 매뉴얼 PDF가 함께 제공되면, 아래 규칙으로 다루지 못하는 구문은 그 매뉴얼을 우선 참고한다. 아래 규칙은 매뉴얼을 바탕으로 확정한 작업 규칙이다.

## 핵심 변환 규칙

아래 규칙들은 모두 "한컴오피스에서 실제로 올바른 높이·간격·서체로 렌더링되게" 하기 위한 것이다. 단순 문법 변환이 아니라 이 의도를 항상 염두에 둔다.

### 1. 괄호는 모두 LEFT / RIGHT

수식으로 표현되는 괄호는 소괄호·중괄호·대괄호 **모두** `LEFT` `RIGHT`를 적용한다. 안에 분수가 있어도 높이가 알맞게 조정되게 하기 위함이다.

- `(x+1)` → `LEFT( x+1 RIGHT)`
- `[a, b]` → `LEFT[ a`,`` b RIGHT]`
- 절댓값·노름의 `|` 도 `LEFT| ... RIGHT|`

**예외:** `{a} over {b}` 처럼 화면에 괄호로 **표시되지 않고** 구문 구조상 묶어주기만 하는 중괄호에는 LEFT/RIGHT를 넣지 않는다.

**띄어쓰기 주의:** `LEFT`/`RIGHT`와 다른 문자가 붙으면 수식 삽입에 문제가 생긴다. 반드시 공백을 둔다.

- 잘못: `xRIGHT`
- 올바름: `x RIGHT`

### 2. & 위치 (괄호와 연산기호 사이)

`&`는 **괄호와 연산기호 사이**에만 들어간다. 괄호가 없으면 연산기호가 있어도 `&`를 넣지 않는다.

규칙:
- 괄호 뒤에 `+ - = * < > <= >=` 가 오면 그 사이에 `&`를 넣는다.
- 즉 연산기호(`+ - * = > < >= <=`) **왼쪽에** `RIGHT)` / `RIGHT}` / `RIGHT]` / `RIGHT|` 가 있으면 그 사이에 `&`를 넣는다.

```
잘못된 예: lim_{x `->`0&+}      x&=1      theta&=0
올바른 예: f`'LEFT( x RIGHT)&=LEFT( x+1 RIGHT)&+5
```

`x=1`, `theta=0` 처럼 괄호가 없는 경우에는 `&`를 넣지 않는다.

### 3. 아래첨자·위첨자는 항상 중괄호로

불필요한 식이 첨자로 딸려 들어가는 것을 막기 위해, 아래첨자와 위첨자는 **모두 중괄호로 감싼다.**

- `a_1` → `a_{1}`
- `x^2` → `x^{2}`
- `a_{i+1}^{2}`

### 4. 함수 표기 (소문자 함수 이름)

`f(x)`, `g(x)` 처럼 **알파벳 소문자 이름의 함수**는 함수 이름과 괄호 사이에 띄움 기호 `` ` ``를 넣는다.

```
f` LEFT( x RIGHT)&=x
g` LEFT( x RIGHT)&=x+1
h` LEFT( x RIGHT)&=2
```

- 함수가 **아닐 때**는 불필요하게 `` ` ``를 넣지 않는다.
  - 잘못된 예: `y`=`x`+1`
  - 올바른 예: `y=x+1`

### 5. 도함수(프라임)

`f'(x)` 같은 도함수는 프라임 기호와 함수 이름 사이에 띄움 기호 `` ` ``를 넣는다.

- `f'(x)` → `f`'LEFT( x RIGHT)`

알파벳에 프라임이 붙은 경우(예: `c'`)는 `c^{prime}`이 아니라 **`c prime`** 으로 쓴다.

### 6. 대문자 이탤릭의 띄움

대문자 이탤릭은 `A`만 띄움 한 번, **나머지 알파벳은 띄움 두 번**을 넣는다.

```
A` LEFT( x RIGHT)
B`` LEFT( x RIGHT)
```

### 7. 순서쌍·구간의 쉼표

`(a, b)`, `[a, b]` 처럼 순서쌍 또는 구간 표현에서 `,`가 들어가면 **앞에 띄움 한 번, 뒤에 띄움 두 번**을 준다.

```
LEFT( a`,``b RIGHT)
LEFT[ a`,`` b RIGHT]
```

### 8. 벡터·선분

- 벡터: `vec{rm AB it}`
- 선분: `bar{rm AB it}`

### 9. 점(point) vs 도형·함수 이름 (로만 vs 이탤릭)

알파벳이 **점**을 나타내면 이탤릭이 아니라 **로만체(`rm ... it`)**로 표기하는 것이 일반적이다.

- `P(c, 0)` (점 P) → `rm P it LEFT(c`,``0 RIGHT)`

알파벳이 점이 아니라 **도형·함수의 이름**으로 쓰이면 이탤릭으로 둔다(`rm it` 불필요).

- 예: `원 C`, `A(x)` → 그냥 `C`, `A` (이탤릭)

### 10. 극한

극한은 화살표를 `RARROW`가 아니라 **`->`**로 쓴다.

- `lim_{x`->`0+}`

### 11. 분수

분수는 `{} over {}` 형식을 사용한다.

- `{x+1} over {2}`

## 작업 흐름

1. 이미지(또는 LaTeX/텍스트)를 보고 텍스트와 수식을 식별한다.
2. 각 수식을 위 규칙대로 HancomEQN 구문으로 변환한다.
3. 수식 경계를 `$`로 감싼다. 텍스트와 공백 없이 붙으면 수식 끝에 `` ` ``를 붙인다.
4. 텍스트는 원문 그대로 두고 수식 자리에 변환 결과를 끼워 넣는다.
5. 전체를 하나의 마크다운 코드블록으로 내보낸다.

## 종합 예시

입력(이미지 내용): `함수 f(x)=x+1에 대하여 f'(x)=1이다.`

출력:

```
함수 $f` LEFT( x RIGHT)&=x+1`$에 대하여 $f`'LEFT( x RIGHT)&=1`$이다.
```

(주의: `f(x)=x+1` 다음에 `에`가 공백 없이 붙으므로 수식 끝에 `` ` ``, `f'(x)=1` 다음에 `이다`가 붙으므로 끝에 `` ` ``.)

## 체크리스트 (출력 전 자가 점검)

- [ ] 모든 표시용 괄호에 `LEFT`/`RIGHT`가 적용되었나? (`over`의 구조용 중괄호는 제외)
- [ ] `LEFT`/`RIGHT` 양옆에 공백이 있나? (`xRIGHT` 금지)
- [ ] `&`가 괄호와 연산기호 사이에만 들어갔나? (괄호 없는 `x=1`에는 `&` 없음)
- [ ] 모든 첨자가 중괄호로 감싸졌나?
- [ ] 소문자 함수 이름에 띄움 `` ` `` 가 들어갔나? 함수가 아니면 안 넣었나?
- [ ] 대문자 이탤릭 띄움 규칙(A는 1번, 나머지는 2번)을 지켰나?
- [ ] 순서쌍/구간 쉼표가 `` `,`` `` (앞 1·뒤 2) 형태인가?
- [ ] 점은 `rm ... it` 로만체, 도형/함수 이름은 이탤릭인가?
- [ ] 극한 화살표가 `->` 인가?
- [ ] 프라임이 `prime` (또는 도함수는 `` `' ``)으로 처리됐나?
- [ ] 수식 경계 `$` 와 공백 없는 경우의 끝 `` ` `` 가 정확한가?
- [ ] 전체가 마크다운 코드블록으로 감싸졌나?
